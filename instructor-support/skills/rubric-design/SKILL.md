# 평가 루브릭 설계 (Rubric Design)

## 목적 / When to use
기대치를 명시해 채점 주관성을 줄이고 평가를 표준화하는 루브릭을 설계한다. 각 준거는 학습목표의 한 구성요소를 분명히 드러내야 한다.

**언제 쓰나:** 과제·프로젝트·수행평가의 채점 루브릭이 필요하거나, 기존 루브릭을 목표에 정렬시켜야 할 때.

## 입력
- 강좌·차시 학습목표(`syllabus-design` 산출물의 목표를 상속)
- 대상 과제/평가의 성격(에세이·프로젝트·발표·캡스톤 등)

## 루브릭 3대 구성요소
- **평가준거(Criteria)** — 무엇을 평가할지. 각 준거는 학습목표의 한 구성요소를 드러낸다
- **수행수준(Performance Levels)** — 몇 단계로 판정할지(예: 미흡–보통–우수, 1~5점)
- **수준별 기술어(Descriptors)** — 각 준거×수준 칸의 구체적·측정 가능 서술. 수준이 올라갈수록 누적, 서술 형식은 평행

상세 → `references/rubric-anatomy.md`

## 절차 (작성 순서)
1. 각 준거가 **어떤 학습목표를 측정하는지** 확인해 정렬 점검(준거→목표·인지수준 매핑)
2. 수행수준 설정
3. 기술어 작성 — **통과(Pass) 먼저**, 이어 최상위(High Distinction), 그다음 미달(Fail) 순 권장
4. **SOLO 분류**를 복잡성이 높아지는 단계 표현 어휘로 활용 → `references/solo-taxonomy.md`
5. 배점과 준거를 학습목표에 맞추어 타당도 확보

산출물은 **준거(행) × 수행수준(열)** 표로 제시하고, 각 준거 옆에 지지하는 목표(G#)·인지수준을 표기한다. 과제 옆에 준거 요약을 함께 노출하도록 안내한다.

**과정 중심 평가 팁:** AI가 대신 만들 수 있는 결과물 과제라면 결과물 대신 과정·성찰을 평가하는 준거(예: 성찰 40%)를 제안한다.

## 입력·출력 예시
```
입력: 목표 G4=[평가] "데이터 분석 결과를 비판적으로 판단할 수 있다" + 기말 보고서
출력: 준거표 —
      | 준거(지지 목표) | 미흡 | 보통 | 우수 |
      | 근거의 타당성(G4·평가) | … | … | … |
      | 대안 고려(G4·평가) | … | … | … |
```
