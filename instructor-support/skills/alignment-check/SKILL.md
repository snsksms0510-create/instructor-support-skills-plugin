# 정렬 점검 (Alignment Check)

## 목적 / When to use
강의계획서 또는 차시 원고/슬라이드를 교수설계 원리로 감사하여, 정렬 오류를 적발하고 구체적 수정안을 담은 점검 보고서를 만든다. 판정은 **✅ 충족 / ⚠️ 보완 권장 / ❌ 정렬 오류** 3단계.

**언제 쓰나:** `syllabus-design`·`lecture-script`·`slide-outline` 산출물의 품질·정렬을 점검하거나, 기존 강의계획서/원고를 감사할 때.

## 입력
- 점검 대상 산출물(강의계획서 또는 차시 원고/슬라이드)
- 대상 종류에 따라 아래 두 모드 중 하나를 적용

## 모드 A — 강의계획서 점검
1. **학습목표 형식** — '~할 수 있다' 형태·측정 가능 동사 사용, 한 목표 한 동사 여부. 위반 목표는 수정안 제시
2. **인지수준 분포** — 목표들의 Bloom 수준 분포표. 하위(기억·이해) 또는 상위(평가·창조)에 쏠렸는지 판정, 대상 학년 대비 적절성
3. **목표 분해** — 강좌 목표가 주차 목표로 빠짐없이 분해되었는가. 어떤 목표도 지지하지 않는 주차, 어떤 주차에서도 다뤄지지 않는 목표를 적발
4. **목표–평가 정렬** — 각 평가가 측정하는 목표·인지수준 일치 여부. 목표는 분석·창조인데 평가가 기억 수준인 오류 적발
5. **운영 방식 근거** — 통상적 기대와 다른 운영(플립러닝·과정중심평가·AI 정책)에 근거가 명시되었는가

## 모드 B — 차시 원고/슬라이드 점검
1. **차시 목표** — 맨 앞에 ABCD·측정 가능 동사로 제시되었는가, 강의계획서의 강좌 목표를 상속하는가(새 목표 생성 여부)
2. **활동의 인지수준 계열** — 이해→창조로 건너뛰지 않고 중간 적용·분석 단계가 있는가
3. **WHERETO 7요소** 충족 여부 표
4. (슬라이드인 경우) **Mayer 원리 준수** — 한 슬라이드 한 메시지·일관성·신호·중복 회피·분절·개인화
5. **형성평가 지점과 과제 평가 준거** 노출 여부

## 절차 (교차 함의 규칙)
거버넌스 §9 교차 함의를 점검 규칙으로 적용한다. 상세 → `references/alignment-rules.md`
- 설계 순서 검증: 목표→평가→활동→도구. 도구·활동이 목표보다 먼저 정해졌으면 경고
- 인지수준 일치: 목표와 평가·활동의 Bloom 수준 불일치 적발
- 실행층 목표 상속: 슬라이드·핸드아웃·문항이 새 목표를 만들지 않았는지
- 사전/사후 경계: 난이도·변별도·신뢰도 등 학생 응답 데이터가 필요한 지표는 사전 점검 대상이 아님을 명시

각 항목을 **표**로 정리하고(항목 | 판정 | 근거 | 수정 제안), 판정은 ✅/⚠️/❌로 표기한다. 불필요한 서론·맺음말은 쓰지 않는다.

## 입력·출력 예시
```
입력: 강의계획서 (목표 G3=[분석]인데 평가는 객관식 암기 위주)
출력: 항목별 ✅/⚠️/❌ 표 →
      "❌ 목표–평가 정렬: G3는 분석 수준이나 중간고사 문항이 기억 수준.
       수정 제안: 사례 분석형 서술 문항으로 대체하거나 G3 수준을 재조정"
```
