# 도구 매핑/가이드 (Tool Mapping & Guide)

## 목적 / When to use
주차별 강의안의 각 활동(TLA)에 어떤 도구를 대응시킬지 결정하고(**도구 매핑**), 매핑된 도구를 어떻게 운영할지 실행 지침을 만든다(**도구 가이드**).

**언제 쓰나:** 활동에 맞는 도구(LMS 기능·협업툴·시뮬레이션·AI 도구·응답시스템)를 고르거나, 도구 사용 안내·운영 팁이 필요할 때.

> 참고: 이 영역은 문헌 근거가 약한 고유 기능이므로, 정렬 원칙에 종속시켜 정의하고 표준 용어처럼 단정하지 않는다.

## 입력
- 주차별 강의안의 활동 목록(TLA)과 각 활동이 지지하는 학습목표·인지수준
- 기관 보유 도구/LMS, Canva 계정 티어(있다면), 접근성·개인정보 요건

## 도구 선택 점검 (활동 학습목표가 도구를 결정)
도구를 먼저 정해 활동을 끼워 맞추지 않는다. 각 도구는 아래를 점검한다.
1. 활동 학습목표(인지수준)에 맞는가
2. 기관 LMS와 연동되는가
3. 학생이 새 계정 없이 쓸 수 있는가(도입 장벽)
4. 접근성(WCAG)·개인정보(FERPA 등) 요건을 충족하는가

도구 수를 늘리는 것 자체가 목적이 되어서는 안 되며, 학생이 익힐 도구는 최소화한다. 활동유형별 대표 도구·LMS 공통 기능군 → `references/tool-catalog.md`

## Canva 중심 매핑 시 주의
- **요금제·권한 먼저 확인**: 자동 생성(Autofill)은 Campus/Enterprise 티어 + 관리자급 권한 전제. 무료·Pro·일반 교수자 계정이면 수동 워크플로로 분기
- 자동 생성 시 **색·폰트 변경 불가** 전제로 템플릿 설계(API로 바꿀 수 있는 것은 텍스트·이미지·차트뿐)
- Canva가 약한 영역(실시간 형성평가·비동기 영상 토론)은 특화 도구로 보완
- 활동→Canva 기능 매핑표 → `references/canva-mapping.md`

## 절차
1. **도구 매핑표** 작성 — 활동(TLA) | 대응 도구 | 지지 학습목표(G#·인지수준) | 선택 근거 | 동기/비동기. 서로 추적 가능하게 연결
2. **도구 가이드** 작성 — 도구별 준비·접속·사용 단계, 교수자용 운영 팁(자주 발생하는 문제와 대비책), 학생에게 그대로 전달할 안내 문구
3. LMS는 공통 기능군(콘텐츠·과제·퀴즈·성적부·소통·분석·LTI 연동)으로 안내하되, 사용자의 LMS를 먼저 확인해 제품 용어로 제시

## 입력·출력 예시
```
입력: 7주차 활동 "조별 A/B 테스트 설계 실습"(G3·적용), 기관 LMS=Canvas, Canva=무료
출력: 매핑표 1행(활동|Miro 화이트보드|G3·적용|실시간 협업 적합|동기) +
      운영 가이드(접속·역할분담·자주 겪는 문제) + Canva 무료라 수동 워크플로 안내
```
