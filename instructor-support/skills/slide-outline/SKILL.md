# PPT 슬라이드 개요 (Slide Outline)

## 목적 / When to use
확정된 강의계획서의 **해당 주차 목표를 상속**하여, Mayer(2009) 멀티미디어 학습 원리를 적용한 슬라이드 개요(아웃라인)를 작성한다. 새 목표를 만들지 않는다.

**언제 쓰나:** 특정 주차 강의용 슬라이드(PPT) 개요·발표자 노트가 필요할 때. 문서형 대본이 필요하면 `lecture-script`를 쓴다.

## 입력
- 확정된 강의계획서(`syllabus-design` 산출물 또는 사용자 제공)
- 대상 주차 번호
- (선택) 교수자 추가 요청

## Mayer 원리 — 슬라이드 점검 규칙으로 적용
- **한 슬라이드 한 메시지**, 텍스트 최소화(불릿 3~5개, 각 12자 내외 권장)
- **일관성**: 학습과 무관한 단어·그림·소리는 넣지 않는다
- **신호**: 슬라이드마다 핵심 강조 포인트를 하나 지정한다
- **공간 근접**: 관련된 글과 그림은 가까이 배치하도록 지시한다
- **중복 회피**: 발표자 노트는 화면 텍스트를 그대로 읽지 않고 부연·예시·발문으로 구성한다
- **분절·사전훈련**: 자료를 소화 가능한 단위로 나누고 핵심 용어는 앞부분에서 미리 소개한다
- **개인화**: 발표자 노트는 대화체로 쓴다

상세 → `references/mayer-principles.md`

## 절차
1. 맨 앞에 **차시 학습목표**(ABCD, 인지수준 태그, 지지하는 강좌 목표 G# 표기) + **슬라이드 구성 개요**(구간별 슬라이드 수·시간 배분 표)
2. 슬라이드별 개요 — 각 슬라이드를 아래 형식으로:

```
### 슬라이드 N — [슬라이드 제목]
- 핵심 메시지(1개): …
- 본문 개요: 불릿 3~5개
- 시각자료 제안: (도표/사진/다이어그램 등 + 배치 지침 — 공간 근접)
- 강조 신호: 이 슬라이드에서 강조할 한 가지
- 발표자 노트(대화체): 화면을 그대로 읽지 않는 부연·예시·발문
```

3. **슬라이드 흐름**: 목표 제시(W) → Hook(H) → 선수지식 연결 → 개념(토대→응용, 분절) → 형성평가 슬라이드(퀴즈/투표 1~2장) → 요약·다음 차시 예고
4. 마지막에 **[Mayer 원리 자가점검]** 표: 각 원리별 준수 여부와 근거 한 줄
5. **사용자 검토 후 Canva 연동 제안**: 개요와 자가점검표를 사용자가 검토·확정하기 전에는 Canva로 넘기지 않는다. 사용자가 개요를 확정하면, Canva 커넥터가 연결되어 있는지 확인하고 "이 개요로 Canva 슬라이드 초안을 만들지" 제안한다. 동의 시 슬라이드별 제목·핵심 메시지·본문 불릿·시각자료 제안을 Canva 커넥터에 전달해 디자인 초안을 생성한다. 커넥터가 연결되어 있지 않으면 개요 문서로 마무리하고, 연결 방법(Claude Desktop/Cowork → Settings → Connectors → Canva)만 안내한다.

## 입력·출력 예시
```
입력: 강의계획서 + "3주차"
출력: 차시 목표 + 슬라이드 구성표 → 슬라이드 1~N 개요(핵심메시지·시각자료·강조·발표자노트)
      → [Mayer 원리 자가점검표] → (사용자 확정 후, Canva 연결 시) Canva 슬라이드 디자인 초안
```

## 참고
- Mayer 원리 상세: `references/mayer-principles.md`
- 점검: 작성 후 `alignment-check`(script 모드)로 목표 상속·Mayer 준수를 감사.
- Canva 연동: 개요 확정 후 Canva 커넥터로 실제 디자인 슬라이드를 생성할 수 있다(대화 내 요청 시에만 동작, 자동화 트리거 없음).
