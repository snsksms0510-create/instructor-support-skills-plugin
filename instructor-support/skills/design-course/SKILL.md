# 교수설계 전체 파이프라인 (Design Course)

## 목적 / When to use
`instructor-support` 플러그인의 스킬들을 **정렬 순서대로 모두 실행**해, 강좌 설계 패키지 전체를 한 번에 산출한다. 개별 스킬을 하나씩 부르는 대신 이걸 쓰면 전 과정이 이어진다.

**언제 쓰나:** "강의 전체를 설계해줘", "이 강의계획서로 원고·평가·도구까지 다 만들어줘"처럼 전 과정을 일괄로 원할 때.

## 입력
아래 중 무엇이든 받는다.
- **강의 기본정보**(과목명·학문분야·대상·학점/시수·총주차·강의방식·주요주제·수강생특성·평가선호). → 1단계부터 시작
- **완성된 강의계획서**(첨부 파일 등). → 1단계는 건너뛰고 2단계부터 시작
- **아무것도 없으면**, 강의 기본정보 항목을 사용자에게 한 번에 되물어 수집한다. 누락 항목은 지어내지 말고 `[입력 필요]`로 둔다.

## 절차 (정렬 원칙: 목표 → 평가 → 활동 → 도구)
각 단계는 해당 스킬의 방법론을 그대로 따른다. 학습목표(동사·인지수준)를 단계 간에 상속시켜 정렬을 유지한다. 각 단계 완료 시 한 줄로 진행 상황을 알린 뒤 다음으로 넘어간다.

1. **강의계획서** — `syllabus-design` 방식으로 강좌·주차 학습목표(ABCD·인지수준 태그), 주차별 일정, 평가 구성, 커리큘럼 맵을 산출. (강의계획서가 이미 입력되면 이 단계는 생략하고 그 문서를 기준으로 삼는다.)
2. **1차 정렬 점검** — `alignment-check`(모드 A)로 목표 형식·Bloom 분포·목표 분해·목표–평가 정렬을 감사. ❌/⚠️ 항목은 수정안을 적용해 계획서를 보정한 뒤 진행.
3. **평가 설계** — `rubric-design`(주요 과제 루브릭) + `assessment-blueprint`(평가 청사진·문항)을 계획서 목표를 상속해 산출. 사전/사후 경계 명시.
4. **차시 자료** — 사용자가 주차를 지정하지 않으면 **초반·중반·기말 각 1개 주차**를 골라, 각 주차마다:
   - `lecture-script`(문서형 차시 원고 + WHERETO 자가점검)
   - `slide-outline`(PPT 슬라이드 개요 + Mayer 자가점검)
   각 산출물은 해당 주차 목표를 상속하고 새 목표를 만들지 않는다.
5. **도구 매핑** — `tool-mapping`으로 차시 활동(TLA)별 도구 매핑 표 + 운영 가이드.
6. **최종 정렬 점검** — 생성된 차시 원고/슬라이드에 `alignment-check`(모드 B)로 목표 상속·인지수준 계열·WHERETO/Mayer 준수를 감사.

> 각 단계의 상세 규칙·프레임워크는 형제 스킬의 `SKILL.md`와 `references/`를 따른다: `syllabus-design`, `alignment-check`, `rubric-design`, `assessment-blueprint`, `lecture-script`, `slide-outline`, `tool-mapping`.

## 산출
아래 구조의 통합 결과를 마크다운으로 제시한다.
```
# {과목명} — 교수설계 패키지
## 1. 강의계획서 (+ 커리큘럼 맵)
## 2. 평가 설계 (루브릭 · 평가 청사진 · 문항)
## 3. 차시 자료 (선정 주차별 원고 · PPT 개요)
## 4. 도구 매핑/가이드
## 5. 정렬 점검 요약 (✅/⚠️/❌ 및 적용한 수정)
```

## 운영 규칙
- 분량이 크므로 단계별로 진행 상황을 알린다. 사용자가 특정 단계만 원하면(예: "평가는 빼줘") 건너뛴다.
- 대량 산출이 부담되면 파일로 저장할지 물어본다.

## 입력·출력 예시
```
입력: syllabus-sample.md 첨부 + "이 강의계획서로 전체 설계해줘"
출력: 1단계 생략 → 정렬 점검(심어둔 오류 3개 적발) → 평가 설계 → 3개 주차 원고·PPT
      → 도구 매핑 → 최종 점검 요약, 하나의 패키지로 통합
```

## 참고
- 개별 단계만 원하면 각 스킬을 직접 호출한다.
- 정렬 원칙 전반: `alignment-check`의 `references/alignment-rules.md`.
