# 평가 청사진 (Table of Specifications / Test Blueprint)

Fives & DiDonato-Barnes(2013) 등. 무엇을 가르쳤는지와 무엇을 평가하는지를 대조하는 표.

## 형식
**내용 영역(행) × 인지수준(열)** 격자에 문항 수·배점을 배치한다.

| 내용 영역 | 기억 | 이해 | 적용 | 분석 | 평가/창조 | 계(문항·배점) |
|---|---|---|---|---|---|---|
| 영역 1 | 2 | 1 | 1 | | | 4 (16점) |
| 영역 2 | 1 | 1 | 2 | 1 | | 5 (20점) |
| 영역 3 | | 1 | 1 | 2 | 1 | 5 (24점) |
| … | | | | | | |

## 원칙
- **내용 타당도**: 수업 강조 비중이 평가에 비례 반영되게 한다.
- **정렬 강화**: 문항이 학습목표·인지수준과 일치하는지 검증한다.
- **쏠림 방지**: 교수자는 문항의 80~90%를 암기 수준에 두는 경향이 있으므로, 청사진으로 인지수준 분포를 의도적·가시적 결정으로 만든다.
- **권장 분포**(학부 시험): 하위수준 30~40% / 상위수준 60~70%. 강좌 수준·목표에 따라 조정.
- 청사진에 채점 기준(규준참조/준거참조)까지 명시하면 완전한 시험 계획이 된다.

## 사전/사후 경계
- **사전 지원 가능**: 출제, 청사진, 문항 작성 원칙.
- **사후(학생 응답 데이터 필요)**: 난이도·변별도·신뢰도 등 문항 분석. 데이터 없이 추정하지 않고, 산출물에 이 경계를 명시한다.
