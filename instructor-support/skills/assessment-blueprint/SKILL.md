# 평가 청사진·문항 출제·채점 (Assessment Blueprint)

## 목적 / When to use
루브릭 설계에서 나아가 실제 평가를 출제·채점하는 단계를 지원한다. **평가 청사진(Table of Specifications)**으로 내용 타당도와 인지수준 정렬을 확보하고, 원칙에 맞는 문항과 채점/피드백 가이드를 만든다.

**언제 쓰나:** 중간·기말 시험 문항을 출제하거나, 무엇을 얼마나 평가할지(내용×인지수준)를 사전 설계해야 할 때.

## 입력
- 강좌·차시 학습목표(상속), 내용 영역 목록, 평가 유형·문항 수·배점 정보

## 평가 청사진 (Table of Specifications)
- 무엇을 가르쳤는지와 무엇을 평가하는지를 대조하는 표. **내용 영역(행) × 인지수준(열)** 격자에 문항 수·배점 배치
- 내용 타당도: 수업 강조 비중이 평가에 비례 반영. 정렬 강화: 문항이 목표·인지수준과 일치하는지 검증
- **쏠림 방지**: 교수자는 문항 80~90%를 암기 수준에 두는 경향이 있으므로, 청사진으로 인지수준 분포를 의도적·가시적으로 결정
- 학부 시험 권장 분포: 하위수준 30~40% / 상위수준 60~70% (강좌 수준·목표에 따라 조정)

상세 → `references/test-blueprint.md`

## 객관식(MCQ) 작성 원칙
- 한 문항은 하나의 학습목표만 측정 / 문항 간 독립성 유지
- 발문(stem)은 완결된 질문 형태(선택지 보기 전 답 떠올릴 수 있게). 빈칸형·부정문(NOT, EXCEPT) 회피, 부득이하면 강조
- 오답(distractor)은 흔한 오개념에서 도출. 우스꽝스럽거나 비현실적 오답 금지
- 정답 단서 제거(문법 a/an·단복수, 길이 단서, 정답이 가장 길지 않게, 선택지 길이·복잡도 균일)
- **오답을 먼저 작성**(정답 먼저 쓰면 오답이 변형돼 단서화). 선택지 보통 4개(오답 3개)
- 'all/none of the above'·복수정답형 회피. 단순 회상에 그치지 않고 해석·추론·인과 설명 등 고차사고 측정

상세 → `references/mcq-rules.md`

## 절차
1. **평가 청사진** 표 산출(내용 영역×인지수준, 문항 수·배점) — 정렬 점검 도구로 활용
2. 청사진에 맞춰 **문항 작성**(문항이 한 목표만·그 인지수준으로 측정하는지 점검)
3. **채점/피드백 가이드** — 피드백은 루브릭 준거에 맞춘 구체적 문구(무엇이·왜·어떻게 개선). 채점자 간·내 일관성을 위한 기준 사례(앵커) 제시

**사전/사후 경계(중요):** 출제·청사진·문항 작성 원칙은 **사전 지원** 가능하나, 난이도·변별도·신뢰도 등 문항 분석은 **학생 응답 데이터가 필요**하다. 이 경계를 산출물에 명시하고, 데이터 없이 추정하지 않는다.

## 입력·출력 예시
```
입력: 목표 G1~G5 + 내용영역 4개 + 기말 25문항
출력: 청사진 표(영역×[기억~창조], 문항수·배점) → MCQ 문항 세트(정답·오답 근거)
      → 채점/피드백 가이드 + "난이도·변별도는 응답 데이터 확보 후 분석" 명시
```
