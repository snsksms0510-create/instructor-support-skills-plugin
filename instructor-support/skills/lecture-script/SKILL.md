# 문서형 차시 원고 (Lecture Script)

## 목적 / When to use
확정된 강의계획서의 **해당 주차 목표를 상속**하여, 실제 수업에서 그대로 쓸 수 있는 차시 원고(강의안 + 강의 대본)를 작성한다. 새 목표를 만들지 않고 계획서의 목표를 지지한다.

**언제 쓰나:** 특정 주차의 수업을 진행할 강의안·대본이 필요할 때(문서형). 슬라이드가 필요하면 `slide-outline`을 쓴다.

## 입력
- 확정된 강의계획서(`syllabus-design` 산출물 또는 사용자 제공)
- 대상 주차 번호
- (선택) 교수자 추가 요청 — 예: 사례 중심, 조별 토론 20분 포함, 동영상 강의용 등

## 핵심 원칙
- 계획서의 강좌 목표(G#)와 주차 목표를 상속한다. **새 목표를 만들지 않는다.**
- 활동의 인지수준은 계열을 이룬다: 도입의 이해에서 과제의 창조로 건너뛰지 않고 **중간의 적용·분석 단계를 명시**한다.
- 개념은 '쉬운 말 + 예시'로 이중 제시하고, 단계적 비계(시범→안내된 연습→자율 적용)를 놓아 인지부하를 분산한다.
- 대본은 대화체(개인화 원리)로 쓴다. 근거가 필요한 결정은 이유를 밝힌다.

## 절차
아래 구조를 **이 순서로** 작성한다.

1. **차시 학습목표** — 계획서에서 분해된 모듈 수준 목표를 맨 앞에. ABCD·측정 가능 동사·인지수준 태그, 어느 강좌 목표(G#)를 지지하는지 표기
2. **선수지식과의 연결** — 새 내용을 기존 지식·이전 차시와 연결하는 고리(선개념 활성화 질문 포함)
3. **도입(Hook)** — 도발적 질문·실제 문제·사례로 내재적 동기 자극 (약 5~10분)
4. **본 활동의 계열** — 활동별 시간 배분 표와 함께, 토대 개념→응용 순으로 배열. 각 활동이 어느 차시 목표를 지지하는지 표기. 이해→창조로 건너뛰지 않도록 적용·분석 단계 명시
5. **강의 대본** — 각 구간별 교수자 발화 흐름을 대화체로. 판서/화면 지시, 발문(질문)과 예상 답변 포함
6. **형성평가 지점** — 차시 안에서 이해를 확인하는 지점 1~2곳(1분 쪽지, 개념 퀴즈 등). 과제가 있으면 평가 준거 요약을 과제 옆에 함께 제시
7. **정리 및 다음 차시 연결** — 핵심 요약(인출연습 형태 권장) + 다음 차시 예고
8. **학습 자료 및 매체** — 읽기 자료·슬라이드·도구 목록

**비동기·플립러닝·동영상 차시**인 경우 추가: 사전개념 활성화 질문, 시청 전후 대조 활동, 학습자 분기 문항(예: 실무자용 별도 문항, UDL 관점), "내 맥락에 어떻게 적용할까" 전이 성찰.

마지막에 **[WHERETO 자가점검]** 표를 붙인다: W/H/E/R/E/T/O 각 요소의 충족 여부와 해당 지점을 한 줄씩. → `references/whereto-checklist.md`

## 입력·출력 예시
```
입력: 강의계획서 + "7주차" + "A/B 테스트, 조별 실습 20분 포함"
출력: 7주차 목표(G3 지지, [적용]) → 선수지식 연결 → Hook(실제 실험 사례)
      → 활동 계열표(개념→실습→해석) → 대본 → 형성평가(개념 퀴즈) → 정리 → [WHERETO 점검표]
```

## 참고
- WHERETO 점검 상세: `references/whereto-checklist.md`
- 점검: 작성 후 `alignment-check`(script 모드)로 목표 상속·인지수준 계열·WHERETO를 감사.
