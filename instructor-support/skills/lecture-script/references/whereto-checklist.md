# WHERETO 점검표

Wiggins & McTighe(2005) 백워드 설계(UbD). **순서가 아니라** 이미 짜놓은 설계가 좋은지 **사후에 검증**하는 분석 도구다.

| 요소 | 세부 내용 | 점검 질문 |
|---|---|---|
| **W** (Where/Why) | 교사가 정한 목표를 학생에게 투명하게 전달. 첫날부터 최종 수행과제·루브릭·본질적 질문 공개, 진단평가로 출발점 점검 | 학생이 목표·평가 기준을 처음부터 알고, 출발점(선행지식)도 파악되는가? |
| **H** (Hook) | 상벌이 아니라 내재적 동기 자극. 도발적 질문·실제 문제·미스터리·역할극으로 "알 가치가 있게" 만들되 본질을 향하게 | 빅 아이디어로 끌어들이고 붙드는가? |
| **E** (Equip/Explore) | 학습 계획의 핵심부. 실제·모의 경험, 스캐폴딩(시범→안내된 연습→자율 적용)으로 비계 점차 제거 | 경험시키고 최종 수행을 위해 갖춰주는가? |
| **R** (Rethink) | 한 번에 숙달되지 않으므로 흐름 반복. 관점·공감 전환으로 초기 이해 거듭 수정 | 재고·성찰·수정 기회가 있는가? |
| **E** (Evaluate) | 메타인지(자기지식)를 명시적으로 가르침. 1분 에세이, 자기평가서, 동료평가 훈련 | 자기평가·진척 점검을 설계했는가? |
| **T** (Tailor) | 목표·이해는 유지하되 내용·과정·결과물 조정. 산출물 선택권을 주되 공통 준거로 평가 | 다양한 학습자에 맞췄는가(목표는 유지)? |
| **O** (Organize) | 여섯 요소를 가장 강력한 순서로 엮음. 전체→부분→전체, 흥미 유발 먼저, 정의·이론은 필요할 때까지 미룸 | 가장 효과적인 순서로 배열했는가? |

## 차시 설계 보조 원칙
- **계열화**: 고립된 사실의 모음이 아니라 하나의 이야기가 되도록 토대→복합 개념으로 점진 전개
- **분산학습·인출연습**: 한 차시에 몰지 않고 분산, 이전 차시 내용을 다시 꺼내는 활동 배치
- **비동기·플립러닝형 차시 보강**: 사전개념 활성화, 시청 전후 대조, 학습자 분기, 전이 성찰 포함
- 원고(강의 전달용 대본)와 강의안(차시 운영 설계)을 구분해 다룬다.
