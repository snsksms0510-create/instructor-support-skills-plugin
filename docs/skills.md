# 스킬 상세

`instructor-support` 플러그인의 7개 스킬. 각 스킬 폴더의 `references/`에 프레임워크 상세(동사표·점검표·규칙)가 있다.

## 핵심 — 강의 설계·실행

| 스킬 | 입력 | 하는 일 | 산출물 |
|---|---|---|---|
| `syllabus-design` | 강의 기본정보(과목·대상·주차·방식·주제·수강생특성·평가선호) | 강의계획서 설계(학습목표 내장) | 강의계획서 11개 구성요소 + ABCD 학습목표(인지수준 태그) + 커리큘럼 맵 |
| `lecture-script` | 강의계획서 + 대상 주차 | 문서형 차시 원고(강의안+대본) | 목표·선수지식·Hook·활동계열·대본·형성평가·정리 + WHERETO 자가점검 |
| `slide-outline` | 강의계획서 + 대상 주차 | Mayer 원리 기반 PPT 개요 | 슬라이드별 핵심메시지·본문·시각자료·강조·발표자노트 + Mayer 자가점검 |
| `alignment-check` | 위 산출물 | 정렬 감사(audit) | ✅/⚠️/❌ 점검 보고(목표 형식·Bloom 분포·목표분해·목표–평가 정렬·운영 근거) + 수정 제안 |

## 확장 — 평가·도구

| 스킬 | 하는 일 | 산출물 |
|---|---|---|
| `rubric-design` | 학습목표에 정렬된 채점 루브릭 (거버넌스 §7) | 준거×수행수준 루브릭 표(수준별 기술어, SOLO 활용) |
| `assessment-blueprint` | 평가 청사진·문항 출제·채점 (§8) | 청사진 표(내용영역×인지수준) + MCQ 문항 + 채점/피드백 가이드 (사전/사후 경계 명시) |
| `tool-mapping` | 활동→도구 매핑·운영 가이드 (§6) | 활동×도구×지지목표 매핑 표 + 도구 운영 가이드(Canva 티어·LMS·접근성) |

## 전체 실행 스킬

`design-course` — 아래 스킬 간 계약 순서대로 전 과정을 한 번에 실행하는 오케스트레이터 스킬. 정의: `instructor-support/skills/design-course/SKILL.md`. 강의 기본정보/완성된 강의계획서/무입력 모두를 받으며, 단계별로 진행 상황을 알리고 특정 단계 건너뛰기·파일 저장을 지원한다.

## 스킬 간 계약 (정렬 순서)

```
강의 기본정보
   → syllabus-design      (강좌·주차 목표 확정, 커리큘럼 맵)
   → alignment-check      (목표·인지수준·정렬 점검)
   → lecture-script / slide-outline   (주차 목표 상속)
   → (필요 시) rubric-design · assessment-blueprint · tool-mapping
```

- **학습목표(동사·인지수준)가 모든 스킬을 잇는 공통 키**다. 스킬 간에 주고받는 단일 객체로 다룬다.
- `syllabus-design`의 주차별 목표가 `lecture-script`/`slide-outline`의 입력이 된다(이 연결이 끊기면 정렬 단절이 차시로 전파).
- 실행층 산출물(슬라이드·핸드아웃·문항)은 새 목표를 만들지 않고 설계층 목표를 상속한다.
