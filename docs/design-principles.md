# 설계 원리

이 플러그인은 교수설계 거버넌스를 규칙으로 옮긴 것이다. 모든 스킬이 아래 원리를 공유한다.

## 정렬(alignment) 순서
목표(의도) → 평가(증거) → 활동(경험) → 도구(실행) 순서로 설계한다. 도구·활동이 목표보다 먼저 정해지면 `alignment-check`가 경고한다.

## 핵심 규칙
- **학습목표는 학습자의 도달점**("학생이 무엇을 할 수 있다")으로, ABCD 모델 + 측정 가능한 단일 동사로 기술. '이해한다·안다' 등 측정 불가 동사 금지.
- **인지수준 일치**: 목표가 분석·창조인데 평가가 기억 수준이면 정렬 오류.
- **강좌 목표 → 주차 목표 분해**가 스킬 간 계약. 강의계획서의 주차 목표가 차시 자료의 입력이 된다.
- **실행층은 설계층 목표를 상속**: 슬라이드·핸드아웃·문항은 새 목표를 만들지 않는다.
- **사전/사후 경계**: 출제·청사진·문항 작성은 사전 지원 가능하나, 난이도·변별도·신뢰도는 학생 응답 데이터가 필요 → 산출물에 명시.
- **세 가지 학습 영역**(인지·정의적·심동적)을 모두 고려. 실기·태도 중심 강좌를 놓치지 않는다.
- **도구는 활동 학습목표가 결정**. 단일 벤더에 종속시키지 않고, 학생이 익힐 도구는 최소화.

## 근거가 되는 교수설계 프레임워크
- ABCD 학습목표 (Heinich et al., 1996)
- Bloom 개정분류 (Anderson & Krathwohl, 2001) — 기억·이해·적용·분석·평가·창조
- 구성적 정렬 (Biggs, 1996)
- 백워드 설계·WHERETO (Wiggins & McTighe, 2005)
- 멀티미디어 학습 원리 (Mayer, 2009)
- SOLO 분류 · 평가 청사진(Table of Specifications)

각 프레임워크의 표·점검표·규칙은 해당 스킬의 `references/` 폴더에 있다.

## 저장소 구조
```
.claude-plugin/marketplace.json     # 마켓플레이스 등록
instructor-support/                 # 플러그인
  .claude-plugin/plugin.json
  skills/<skill>/
    SKILL.md
    references/                     # 프레임워크 상세
samples/                            # 입력/산출 샘플
docs/                               # 문서 (이 폴더)
```

## 기여 규약
- 스킬 폴더/id는 영문 kebab-case, 본문은 한국어
- SKILL.md frontmatter에 `name`·`description` 필수
- 본문 ≤ 500줄, 긴 표·프레임워크는 `references/`로 분리
